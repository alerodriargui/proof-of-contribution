# Security Policy

## Supported Versions

Only the latest release or the code on the `main` branch is actively supported and receives security updates.

| Version | Supported |
| ------- | --------- |
| main    | :white_check_mark: |
| < dev   | :x:       |

## Reporting a Vulnerability

We take the security of this project seriously. If you find a security vulnerability, please do not open a public issue. Instead, report it privately using one of the following methods:

1. **GitHub Private Vulnerability Reporting**: If available, navigate to the **Security** tab of this repository on GitHub, click **Vulnerability reporting**, and click **Report a vulnerability**.
2. **Email**: Send a detailed report to **[alerodriargui@gmail.com](mailto:alerodriargui@gmail.com)**.

### What to Include

To help us investigate and resolve the issue quickly, please include:
- A description of the vulnerability and its potential impact.
- Step-by-step instructions (or a proof-of-concept script/exploit) to reproduce the behavior.
- Details of the environment where it was tested.

### Our Response Process

After receiving your report:
- We will acknowledge receipt of your report within 48 hours.
- We will investigate and attempt to reproduce the issue.
- Once confirmed, we will work on a fix and coordinate the release date with you.
- We ask that you keep the report confidential until we have published a fix.
